# Computer Science Top Conference Submission Guide

## Overview

This guide covers submission requirements and best practices for top-tier computer science conferences: NeurIPS, ICML, ICLR, KDD, and their equivalents.

## Conference-Specific Guidelines

### NeurIPS (Neural Information Processing Systems)

**Conference Focus**: Machine learning, computational neuroscience, AI

**Submission Requirements**:
- **Page Limit**: 8 pages + unlimited references
- **Format**: LaTeX using NeurIPS template
- **Font Size**: Minimum 10pt
- **Anonymity**: Double-blind review
- **Supplementary Material**: Allowed (code, proofs, additional experiments)
- **Reproducibility**: Encouraged to submit code and reproducibility checklist

**Review Criteria**:
1. Technical correctness and soundness
2. Novelty and significance
3. Clarity and presentation
4. Relevance to the NeurIPS community
5. Reproducibility and ethical considerations

### ICML (International Conference on Machine Learning)

**Conference Focus**: Machine learning theory and applications

**Submission Requirements**:
- **Page Limit**: 8 pages + unlimited references
- **Format**: LaTeX using ICML template
- **Anonymity**: Double-blind review
- **Reproducibility**: Reproducibility checklist required

**Review Criteria**:
1. Novelty and significance
2. Technical quality and correctness
3. Clarity and reproducibility
4. Relevance to ICML community
5. Broader impact and ethical considerations

### ICLR (International Conference on Learning Representations)

**Conference Focus**: Deep learning, representation learning

**Submission Requirements**:
- **Page Limit**: 8 pages + unlimited references
- **Anonymity**: Double-blind review
- **Supplementary Material**: Encouraged
- **Open Peer Review**: Reviews and author responses are public

**Unique Features**:
- Open review process
- Post-conference shepherding for major revisions
- Strong emphasis on code release and reproducibility

### KDD (ACM SIGKDD Conference on Knowledge Discovery and Data Mining)

**Conference Focus**: Data mining, knowledge discovery, big data analytics

**Submission Requirements**:
- **Page Limit**: 10 pages including figures (references unlimited)
- **Format**: ACM Master Template (two-column format)
- **Anonymity**: Double-blind review

**Review Criteria**:
1. Originality and significance
2. Technical quality and soundness
3. Relevance and impact to the KDD community
4. Clarity and presentation
5. Empirical evaluation and reproducibility

## Structure of CS Conference Papers

**Standard Structure**:
1. **Abstract** (150-250 words)
2. **Introduction** (1-1.5 pages)
3. **Related Work** (1-2 pages)
4. **Method/Approach** (2-3 pages)
5. **Experiments** (2-3 pages)
6. **Discussion/Conclusion** (0.5-1 page)
7. **References** (unlimited pages)
8. **Appendix/Supplementary Material** (optional)

## Writing Style for CS Conferences

### Mathematical Notation

Define all variables clearly at first use, use consistent notation, and prefer clarity over brevity.

### Experimental Writing

**Required Elements**:
1. Datasets: Describe size, characteristics, splits
2. Baselines: Include state-of-the-art methods
3. Metrics: Define all evaluation metrics
4. Statistical Significance: Report confidence intervals
5. Ablation Studies: Isolate contribution of each component
6. Hyperparameters: List values and selection method

## Common Mistakes to Avoid

1. Lack of Novelty: Incremental improvements without significant insights
2. Insufficient Baselines: Not comparing with SOTA methods
3. Weak Experimental Evaluation: Limited datasets, no statistical testing
4. Missing Ablation Studies: Unclear which components contribute
5. Overclaiming: Making statements not supported by experiments
